Function¶

  • Function name - Start with [a-z, _] and contains [a-z][0-9][_]
  • May contain other functions and class definitions.
  • Functions need not be defined before they are referenced.
  • Function names are case-insensitive.
  • If the return is omitted the value NULL will be returned.

Passing arguments by reference¶

By default, function arguments are passed by value (so that if the value of the argument within the function is changed, it does not get changed outside of the function). To allow a function to modify its arguments, they must be passed by reference.

1
2
3
4
5
6
7
<?php
function add_some_extra(&$string) {
    $string .= 'and something extra.';
}
$str = 'This is a string, ';
add_some_extra($str);
echo $str; // 'This is a string, and something extra.'

Default argument values¶

1
2
3
4
5
6
7
<?php
function makecoffee($type = "cappuccino") {
    return "Making a cup of $type.\n";
}
echo makecoffee(); // Making a cup of cappuccino.
echo makecoffee(null); // Making a cup of .
echo makecoffee("espresso"); // Making a cup of espresso.

Note that when using default arguments, any defaults should be on the right side of any non-default arguments; otherwise, things will not work as expected

Type declarations¶

Type declarations allow functions to require that parameters are of a certain type at call time. If the given value is of the incorrect type, then an error is generated: in PHP 5, this will be a recoverable fatal error, while PHP 7 will throw a TypeError exception.

<?php
class C {}
class D extends C {}

// This doesn't extend C.
class E {}

// type declaration
function f(C $c) {
    echo get_class($c)."\n";
}

f(new C); // C
f(new D); // D
f(new E); // Uncaught TypeError: Argument 1 passed to f()
          // must be an instance of C, instance of E given

Strict typing¶

By default, PHP will coerce values of the wrong type into the expected scalar type if possible.

1
2
3
4
5
6
7
8
9
<?php
function sum(int $a, int $b) {
    return $a + $b;
}

var_dump(sum(1, 2)); // int(3)

// These will be coerced to integers: note the output below!
var_dump(sum(1.5, 2.5)); // int(3)

    return $a + $b;
}

var_dump(sum(1.5, 2.5)); // Uncaught TypeError: Argument 1 passed to sum() must
                        // be of the type integer, float given

  • ... token in PHP 5.6 and later.
  • func_num_args(), func_get_arg(), and func_get_args() functions in PHP 5.5 and earlier.

function sum(...$numbers) {
    $acc = 0;
    foreach ($numbers as $n) {
        $acc += $n;
    }
    return $acc;
}

echo sum(1, 2, 3, 4); // 10

# ... token during function call
function add($a, $b) {
    return $a + $b;
}

echo add(...[1, 2]); // 3

Return type declarations¶

PHP 7 adds support for return type declarations. Similarly to argument type declarations, return type declarations specify the type of the value that will be returned from a function. The same types are available for return type declarations as are available for argument type declarations.

Strict typing also has an effect on return type declarations. In the default weak mode, returned values will be coerced to the correct type if they are not already of that type. In strong mode, the returned value must be of the correct type, otherwise a TypeError will be thrown.

1
2
3
4
5
6
<?php
function sum($a, $b): float {
    return $a + $b;
}

var_dump(sum(1, 2)); // float(3)

Variable functions¶

This means that if a variable name has parentheses appended to it, PHP will look for a function with the same name as whatever the variable evaluates to, and will attempt to execute it.

<?php
function foo() {
    echo "In foo()";
}

$func = 'foo';
$func(); // $func variable evaluate to `foo`, so it calls foo()

# class
class Foo {
    function Variable() {
        $name = 'Bar';
        $this->$name(); // This calls the Bar() method
    }

    function Bar() {
        echo "This is Bar";
    }
}

$foo = new Foo();
$funcname = "Variable";
$foo->$funcname();  // This calls $foo->Variable()

# callables
class Foo {
    static function bar() {
        echo "bar\n";
    }
    function baz() {
        echo "baz\n";
    }
}

$func = array("Foo", "bar");
$func(); // "bar"
$func = array(new Foo, "baz");
$func(); // "baz"
// as of PHP 7.0.0
$func = "Foo::bar";
$func(); // "bar"

<?php
# In function parameter
echo preg_replace_callback('~-([a-z])~', function ($match) {
    return strtoupper($match[1]);
}, 'hello-world');
// outputs helloWorld

# In variable assignment
$greet = function($name) {
    printf("Hello %s\r\n", $name);
};

$greet('World'); // Hello World

<?php
$message = 'hello';
$example = function () use ($message) {
    return $message;
};
$example(); // hello

# Own arguments and variable from parent scope
$message = 'world';
$example = function ($arg) use ($message) {
    return ($arg . ' ' . $message);
};
$example("hello"); // hello world

Inheriting variables from the parent scope is not the same as using global variables. Global variables exist in the global scope, which is the same no matter what function is executing. The parent scope of a closure is the function in which the closure was declared (not necessarily the function it was called from). See the following example:

function getTotal($tax) {
    $total = 0.00;

    $callback = function ($quantity, $product) use ($tax, $total) {
        // $tax, $total from parent scope
        //..code
    };
}
