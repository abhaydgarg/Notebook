Namespaces¶

Namespaces in very simple words

In the broadest definition namespaces are a way of encapsulating items. This can be seen as an abstract concept in many places. For example, in any operating system directories serve to group related files, and act as a namespace for the files within them. As a concrete example, the file foo.txt can exist in both directory /home/greg and in /home/other, but two copies of foo.txt cannot co-exist in the same directory. In addition, to access the foo.txt file outside of the /home/greg directory, we must prepend the directory name to the file name using the directory separator to get /home/greg/foo.txt. This same principle extends to namespaces in the programming world.

Note: Namespace names PHP and php, and compound names starting with these names (like PHP\Classes) are reserved for internal language use and should not be used in the userspace code.

Defining namespace¶

Although any valid PHP code can be contained within a namespace, only the following types of code are affected by namespaces: classes, interfaces, functions and constants.

<?php
namespace MyProject;

const CONNECT_OK = 1;
class Connection { /* ... */ }
function connect() { /* ... */ }

# Sub namespace
namespace MyProject\Sub\Level;

const CONNECT_OK = 1;
class Connection { /* ... */ }
function connect() { /* ... */  }

// creates constant MyProject\Sub\Level\CONNECT_OK,
// class MyProject\Sub\Level\Connection
// and function MyProject\Sub\Level\connect.

Note: namespace must declare the namespace at the top of the file before any other code, even whitespace - with one exception: the declare keyword.

Multiple namespaces¶

It is allowed to define multiple namespaces in one file, but it is recommended to use one file for single namespace.

Global namespace¶

File without namespace definition, then all class and function definitions are placed into the global space - as it was in PHP before namespaces were supported.

Prefix name with \ = From global space

<?php
namespace A\B\C;

// This function is A\B\C\fopen
function fopen() {
     $f = \fopen(...); // call global fopen, prefix with `\`
     return $f;
}

namespace keyword vs __NAMESPACE__¶

PHP provides two ways to access current namespace.

[1] __NAMESPACE__ constant

<?php
//### 1. namespaced file - when file has namespace specified
namespace MyProject;

echo '"', __NAMESPACE__, '"'; // outputs "MyProject"

//### 2. un-namespaced file (in global) - when file has no namespace specified
// it returns empty string
echo '"', __NAMESPACE__, '"'; // outputs ""

Useful when creating namespace dynamically.

new namespace\MyClass; // MyProject\MyClass

namespace\cname::method(); // calls static method of class MyProject\cname
$b = namespace\CONSTANT; // assigns value of constant MyProject\CONSTANT to $b

//### 2. un-namespaced file (in global) - when file has no namespace specified
// it returns empty string
namespace\func(); // calls function func()
namespace\cname::method(); // calls static method "method" of class cname
$b = namespace\CONSTANT; // assigns value of constant CONSTANT to $b

use keyword¶

It is used to import external fully qualified namespaces.

Note: Must be Fully Qualified (absolute) name and not processed relative to the current namespace.

use has nothing to do with current namespace. It is just for importing other namespace's items to make item's name short when using them.

<?php
namespace foo;

use My\Full\Classname as Another;
use My\Full\NSname;
use function My\Full\functionName;
use const My\Full\CONSTANT;

// importing a global class
use ArrayObject;

$obj = new Another; // instantiates object of class My\Full\Classname
new NSname\MyClass; // calls My\Full\NSname\MyClass
$a = new ArrayObject(array(1)); // instantiates object of global class ArrayObject
                                // without the "use ArrayObject" we would instantiate an
                                // object of class foo\ArrayObject
functionName(); // calls function My\Full\functionName
echo CONSTANT; // echoes the value of My\Full\CONSTANT
