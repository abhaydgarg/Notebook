Language¶

Boolean¶

Consider false

  • the integer 0
  • the float 0.0
  • the empty string
  • the string "0"
  • an array with zero elements
  • the special type NULL (including unset variables)

Every other value is considered TRUE (including any resource and NAN).

Variable parsing¶

When variable is embedded in a string.

Simple syntax¶

If a dollar sign ($) is encountered, the parser will greedily take as many tokens as possible to form a valid variable name.

Enclose the variable name in curly braces to explicitly specify the end of the name.

Double quotes only. Enclosing string in single quotes won't parse variable.

<?php
$juice = "apple";

// Output: He drank some apple juice.
// PHP encounters $ now it start parsing variable name greedly
// and hit space after e, so it stops and identify $juice
// and replace it with the value.
echo "He drank some $juice juice.";

// Output: He drank some juice made of .
// here PHP greedly identify $juices which does not exist in a scope
echo "He drank some juice made of $juices.";

// Output: He drank some juice made of apples.
// Use of {} to explicitly tells PHP where to end
// collecting characters greedly
echo "He drank some juice made of ${juice}s.";

Complex syntax¶

Simply write the expression the same way as it would appear outside the string, and then wrap it in { and }.

Double quotes only. Enclosing string in single quotes won't parse variable.

Note: There must not be any character or space other than $, immediately after {.

// Won't work
// Outputs: This is { fantastic}
// because there is space between { and $

// Works
// Outputs: This is fantastic
echo "This is {$great}";

<?php
$array = array(
    "foo" => "bar",
    "bar" => "foo",
);

// as of PHP 5.4
$array = [
    "foo" => "bar",
    "bar" => "foo",
];

The key can either be an integer or a string.

If PHP does not find valid key which must be either integer or string then, it try to cast it

  • Strings containing valid decimal integers, unless the number is preceded by a + sign, will be cast to the integer type. E.g. the key "8" will actually be stored under 8. On the other hand "08" will not be cast, as it isn't a valid decimal integer.
  • Floats are also cast to integers, which means that the fractional part will be truncated. E.g. the key 8.7 will actually be stored under 8.
  • Bools are cast to integers, too, i.e. the key true will actually be stored under 1 and the key false under 0.
  • Null will be cast to the empty string, i.e. the key null will actually be stored under "".
  • Arrays and objects can not be used as keys. Doing so will result in a warning: Illegal offset type.

If multiple elements in the array declaration use the same key, only the last one will be used as all others are overwritten.

Always use quotes around string key¶

For example, $foo['bar'] is correct, while $foo[bar] is not. But why?

Do not quote keys which are constants or variables, so that PHP can interpret them. But you have to quote string key. Because if you do not then PHP has to do extra work to identify the actual key because without quotes PHP think it must be constant. For example:

// quotes and fruit becomes 'fruits', which works but with extra

// Notice: Use of undefined constant fruit - assumed 'fruit' in
print $arr[fruit];    // apple

// ### lets define constant
define('fruit', 'veggie');
$arr = array('fruit' => 'apple', 'veggie' => 'carrot');

print $arr[fruit];    // carrot not apple

$arr = array('fruit' => 'apple', 'veggie' => 'carrot');
unset($arr);
var_dump($arr); // NULL

Type casting¶

The casts allowed are:

  • (int), (integer) - cast to integer
  • (bool), (boolean) - cast to boolean
  • (float), (double), (real) - cast to float
  • (string) - cast to string
  • (array) - cast to array
  • (object) - cast to object
  • (unset) - cast to NULL
