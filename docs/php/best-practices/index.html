Best practices¶

Variable declaration¶

Always declare variables first and then initialize with default value to avoid "Notice" warning.

Do not use close tag ?> in pure PHP file¶

If there is no ?> closing tag then the last statment must end with semicolon. If there is closing tag then you can omit semicolon from the last statement.

1
2
3
<?php print 'Hello' ?> // omitted ;

<?php print 'Hello'; // cannot omit ; Closing tag is not there.

Never use $_REQUEST¶

set register_global = off¶

true, false and null case¶

Must be in lower case.

For example: true, false, and null must be in lower case.

In PHP:

1
2
3
4

8
<?php
true === TRUE

false === FALSE

null === NULL

// recommended to use in lowercase

Quotes¶

  1. Always use single quotes. Single quotes do not read special character. It print them as it is.
  2. Use double quotes where you want PHP to read and manipulate special characters such as "\n" or variable "$a".
